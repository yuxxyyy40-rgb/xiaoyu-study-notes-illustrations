# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 黑白为主，没有大面积彩色。
- 有小鱼。
- 小鱼是戴圆眼镜、头上套包裹式鱼头帽的理科小人。
- 鱼头帽有圆润帽壳、前额两个小鱼眼睛、侧鳍、圆钝顶部小鳍和少量弧形分段纹路，且不遮住眼镜。
- 小鱼承担核心学习动作，不只是装饰。
- 画面松弛、安静、清楚。
- 主体不超过画面约 55%。
- 至少约 40% 留白。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 中文标注少、短、能读。
- 没有复刻原仓库小黑案例构图。
- 没有默认复用“坐在笔记本上整理卡片”构图，除非主题真的需要。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有“知识结构图 / 学习路径 / 流程图”等标题。
- 小鱼像幼儿贴纸、吉祥物海报或表情包。
- 小鱼像紧张刷题、鸡血学习或演讲状态。
- 小鱼被画成真实鱼或水族馆场景。
- 鱼头帽不够可爱，尤其是缺少前额小鱼眼睛或眼睛太死板。
- 鱼头帽变成毛绒玩偶、动物脸头套或复杂 cosplay。
- 画面像 PPT、课程课件、正式流程图。
- 画面只是普通笔记本和卡片，缺少为当前主题重新发明的物理隐喻。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太紧张、太正式、太商业。
- 彩色喧宾夺主。

## 迭代方法

- 太普通：让小鱼亲自完成一个更具体的物理动作，比如称重、守门、拧旋钮、修补、铺路、回收。
- 太复杂：删节点，只保留一个动作和 3-4 个短标注。
- 太幼稚：保留可爱的鱼头帽，但让脸、身体和动作回到 calm、relaxed、study-note sketch。
- 不够可爱：优先调整鱼帽前额的小鱼眼睛，可以用小圆点眼或困困短弧眼；再轻微增强圆润帽壳。
- 太 PPT：去掉标题、网格、规则箭头和整齐模块，改成低科技物理场景。
- 太彩色：回到纯黑白，只保留必要的浅灰或淡蓝。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该像学习笔记里的一个轻松记忆钩子：第一眼不压迫，第二眼能帮人记住。

如果第一眼像课件页，而不是页边手绘笔记，就不合格。
